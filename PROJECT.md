## Архитектура

Mock-data Flutter mobile прототип для исследования сакральных мест, петроглифов, источников и культурных маршрутов Кыргызстана.

**Структура:**
- `lib/main.dart` — точка входа, роутинг, провайдеры (6658 строк, монолит)
- `lib/app_localizations.dart` — локализация (RU/KG/EN)
- `lib/localized_content.dart` — контент на трёх языках
- `lib/local_ai.dart` — мок-AI гид (Аташка / Апашка)
- `lib/place_image.dart` — изображения мест

**Паттерн:** Riverpod + GoRouter. Без бэкенда — все данные моковые.

---

## Принятые решения и почему

1. **Монолитный main.dart (6658 строк)** — хакathon, скорость важнее архитектуры
2. **Mock-данные вместо API** — прототип, нет бэкенда
3. **Riverpod** — лёгкий state management без boilerplate
4. **Custom painters** — кыргызские визуальные мотивы без внешних ассетов
5. **3 языка (RU/KG/EN)** — целевая аудитория
